# Contributing

Thanks for considering it. This pack is CC0 — by contributing you agree your
work is released into the public domain under the same terms.

**Using an AI agent?** Read [`AGENTS.md`](AGENTS.md) instead; it's the same
information in a form agents follow reliably.

## The one thing to understand first

Every mesh here is generated by script. There are no hand-modelled files, and
`exports/`, `previews/` and `source/ModKit.blend` are all **build outputs**.

To change an asset you change the generator and re-run it. A PR that edits an
FBX directly will be closed, not because it's unwelcome but because the next
build would overwrite it.

## Setup

You need Blender 5.x. Nothing else — no pip installs, no build system. All
scripts use Blender's bundled Python.

```bash
git clone https://github.com/JaronKBragg7337/asset-pack-ue-threejs-blender-unity.git
cd asset-pack-ue-threejs-blender-unity
blender -b --factory-startup --python scripts/build_kit.py
```

That takes a couple of minutes and rewrites all three export formats.

## Adding an asset

1. Write a function in `scripts/build_kit.py`.
2. Add it to `define_assets()`.
3. Rebuild and check it appears in the output.

```python
def roof_flat(name, width=M, depth=M, mat="M_Concrete"):
    """Flat roof panel with a drip lip. Origin at grid corner."""
    bm = bmesh.new()
    L.add_box(bm, (0, 0, 0), (width, depth, cfg.FLOOR_THICKNESS))
    L.weld(bm, (-0.04, -0.04, 0), (width + 0.04, depth + 0.04, 0.06))
    return L.finish(bm, name, material=MATS[mat],
                    collection=L.get_collection("Kit"))
```

Always end with `L.finish()` — it applies the bevel and weighted-normal pass
that gives the pack its consistent look.


## House rules

**No hard-coded dimensions.** Everything lives in `scripts/kit_config.py`.
This is what lets people re-roll the pack at 2 m or 8 m modules. If you need a
new measurement, add a named constant there.

**One watertight manifold per mesh.** Build with `L.add_box()` then `L.weld()`
to union parts together. Loose abutting boxes leave interior faces and break
the bevel into disconnected fragments — the edges stop reading as solid.

**Respect the pivots:**

* Kit pieces → origin at the grid corner (min X, min Y, min Z)
* Floor/ceiling slabs → origin at the corner of the top surface
* Props → origin at base centre

**Stay on the 4 m grid** for anything modular. A piece that doesn't line up
with the existing walls isn't usable, however nice it looks alone.

**No textures.** Shading here is geometric — bevels plus weighted normals, no
normal-map bakes. It's why the pack costs nothing in texture memory.

## Before you open a PR

```bash
python scripts/test_config.py                                   # exits 0
blender -b --factory-startup --python scripts/build_kit.py      # no Traceback
git diff docs/asset_manifest.csv                                # only your asset
```

That last check matters: if triangle counts changed for assets you didn't
touch, you've modified shared geometry somewhere. Worth understanding before
you ship it.

Screenshots or a preview render in the PR are very welcome.
